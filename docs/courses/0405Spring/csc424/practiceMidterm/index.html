Practice Midterm Exam

You will have 65 minutes for this exam. This exam is open book and note. Please allow some time to check your work. If you need extra space, write on the back. There are a total of 50 points on this exam.

  1. Consider the following grammar for a restricted subset of C++ variable declarations:

    Declaration Type  Declarator  ;
    Type int
      | char
    Declarator *  Declarator
      | Declarator  [  Num  ]
      | (  Declarator  )
      | Var

    Assume the usual definitions of Num and Var (the SimpLan definitions are on the last page).
    1. Show two different parse trees for the declaration char *a[10]; [5 points]

    2. Modify the grammar to make it unambiguous, so that char *a[10]; has its usual interpretation (that is, it should be more like char *(a[10]); than char (*a)[10];) [5 points]

  2. Modula-2 solved the dangling-else problem of Pascal by adding an explicit end keyword; to avoid a pile-up of ends after a cascade of nested ifs, it also added an elsif clause: 
    if test1 then
    body1
elsif test2 then
    body2
elsif test3 then
    body3
else
    body4
end
    There may be zero or more elsif clauses, and the final else clause is optional.
    1. Write grammar rules which will allow the non-terminal IfStmt to expand to a Modula-2 if statement, assuming that the non-terminals BoolExpr and StmtSequence are already defined to produce the test expressions and clause bodies. [4 points]

    2. If SimpLan were extended with a similar if statement, show how the above example could be translated into virtual machine code (in place of the translations of the tests and clause bodies, just write [[ test1 ]], [[ body1 ]], etc.; see the last page for a summary of SimpLan) [4 points]

  3. A typical compiler works in several phases: lexical analysis, syntactic analysis, semantic analysis and intermediate code generation, optimization, and object code generation. List some advantages of separating each of these phases. [5 points]

  4. For each of the following binding times, give an example of some attribute in C++ that is bound at that point [2 points each]:
    1. Language definition time
    2. Implementation time
    3. Compile time
    4. Link time
    5. Run time

  5. The following is a legal statement in C++: 
    for (int i = 10; i > 0; (cout << "Hello" << endl), i++) i--;
    Critique the design decisions behind this, listing as many good or bad aspects as you can. [5 points]

  6. Consider the following function skeleton in C++ (none of the omitted statements declare variables): 
    void f() {
    int a, b, c;
    ...
    for (int d = a; d < b; d++) {
        int b, c;
        ...
    }
    if (b == c) {
        int c, d, e, f;
        ...
    } else {
        int g, h, i;
        ...
    }
}
    If an int takes 4 bytes of memory, what is the minimum amount of variable storage space that must be allocated for each call to f, and where is that memory located? [3 points]

  7. Consider the following code in a language somewhat like C++: 
    int x;

void setx(int n) { x = n; }

void f() {
    setx(1);
    cout << x << ", ";
}

void g() {
    int x = 0;
    setx(2);
    cout << x << ", ";
}

int main() {
    f();
    g();
    cout << x << endl;
}
    1. What is the output if the language uses static scoping? [3 points]
    2. What is the output if the language uses dynamic scoping? [3 points]
