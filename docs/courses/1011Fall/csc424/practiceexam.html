CSC 424: Programming Languages, Fall 2010

Practice Midterm

  1. Consider the following grammar for a restricted subset of C++ variable declarations: 
    <Declaration> ::= <Type> <Declarator> ;
       <Type> ::= int | char
 <Declarator> ::= * <Declarator>
                | <Declarator> [ <Num> ]
                | ( <Declarator> )
                | <Var>
    Assume the usual definitions of <Num> and <Var>.
    1. (5 points) Show two different parse trees for the declaration char *a[10];
    2. (5 points) Modify the grammar to make it unambiguous, so that char *a[10]; has its usual interpretation (that is, it should be more like char *(a[10]); than char (*a)[10];).

  2. Modula-2 solved the dangling-else problem of Pascal by adding an explicit end keyword; to avoid a pile-up ofends after a cascade of nested ifs, it also added an elsif clause:

    if test1 then
    body1
elsif test2 then
    body2
elsif test3 then
    body3
else
    body4
end

    There may be zero or more elsif clauses, and the final else clause is optional.

    (4 points) Write grammar rules which will allow the non-terminal <IfStmt> to expand to a Modula-2 if statement, assuming that the non-terminals <BoolExpr> and <StmtSequence> are already defined to produce the test expressions and clause bodies.

  3. (5 points) A typical compiler works in several phases: lexical analysis, syntactic analysis, semantic analysis and intermediate code generation, optimization, and object code generation. List some advantages of separating each of these phases
  4. (2 points each) For each of the following binding times, give an example of some attribute in C++ that is bound at that point:
    • Language definition time
    • Implementation time
    • Compile time
    • Link time
    • Run time
  5. (5 points) The following is a legal statement in C++: 
    for (int i = 10; i > 0; (cout << "Hello" << endl), i++) i--;
    Critique the design decisions behind this, listing as many good or bad aspects as you can.
  6. (3 points) Consider the following function skeleton in C++ (none of the omitted statements declare variables): 
    void f() {
    int a, b, c;
    ...
    for (int d = a; d < b; d++) {
        int b, c;
        ...
    }
    if (b == c) {
        int c, d, e, f;
        ...
    } else {
        int g, h, i;
        ...
    }
}
    If an int takes 4 bytes of memory, what is the minimum amount of variable storage space that must be allocated for each call to f, and where is that memory located?
  7. Consider the following code in a language somewhat like C++: 
    int x;

void setx(int n) { x = n; }

void f() {
    setx(1);
    cout << x << ", ";
}

void g() {
    int x = 0;
    setx(2);
    cout << x << ", ";
}

int main() {
    f();
    g();
    cout << x << endl;
}
    1. (3 points) What is the output if the language uses static scoping?
    2. (3 points) What is the output if the language uses dynamic scoping?
  8. (2 points each; based on Exercise 8.5 of Webber) For each of the following ML function definitions, predict what type ML will infer for f. For example, given the function definition fun f(x) = 1; the answer should be 'a -> int (the actual name of the type variable doesn't matter). 
    1. fun f(x, y) = x;
			
    2. fun f(g) = g(1);
			
    3. fun f(g, x) = g(x + 1);
			
    4. fun f(g, h, x) = g(x) + h(g(x));
  9. (4 points; Exercise 9.4 of Webber) Write an ML function multpairs of type (int * int) list -> int list that takes a list of pairs of integers and returns a list of the products of each pair. For example, if the input is [(1,2), (3,4)], your function should return [2, 12].
  10. (6 points; Exercise 9.23 of Webber) Write a function convert of type ('a * 'b) list -> 'a list * 'b list that converts a list of pairs into a pair of lists, preserving the order of the elements. For example, convert [(1,2), (3,4), (5,6)] should evaluate to ([1, 3, 5], [2, 4, 6]).
