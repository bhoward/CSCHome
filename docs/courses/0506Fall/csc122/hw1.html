CSC 122: Computer Science II, Fall 2005

Homework 1

Due Friday, October 14, in class

  1. Calculate the running time for this code fragment, counting each line of code. 
    1  for (int i = 1; i <= 4; i++)
2    for (int j = 1; j <= 5; j++)
3      System.out.println( i + j );
4  for (int k = 0; k < 25; k++)
5    System.out.println( “Hello” );
  2. Calculate the running time of this code fragment, using the critical step method, assuming that the println statement is the critical step. Do calculations for n = 1, 2, 5, and then a general answer for any n. Then give the O( ) running time. 
    1  int i = 1;
2  while ( i < n) {
3    for (int j = 1; j <= n; j++)
4      System.out.println( “Homer” );
5    i = i + 2;
6  }
  3. Calculate the running time of this code fragment, using the critical step method, assuming that the println statement is the critical step. Do calculations for n = 1, 4, 7, 9, and 16, as well as a general answer for any n. Then give the O( ) running time. 
    1  while (n > 1) {
2	   System.out.println( n );
3	   n = n / 2;
4  }
  4. Consider the following situation. You are given an unsorted array containing N items. You are asked to perform K searches on the array. Your options are to sort the array and perform binary searches, or perform linear searches on the unsorted array. Assume that the sort take N lg N operations to complete, the binary search takes lg N comparisons on average, and that the linear search takes N / 2 comparisons on average. How many searches do you have to do (in other words, what value do you need for K) to make it faster to sort and use binary search than to use linear search for the following values of N?
    1. N = 128
    2. N = 512
    3. N = 2,048
    4. N = 1,000,000 (lg 1,000,000 is approximately 20)
    5. N = 128,000,000 (lg 128,000,000 is approximately 27)
  5. Consider the following Java method: 
    public void enigma( String s ) {
  if ( s.length( ) > 0 ) {
    System.out.println( s );
    enigma( s.substring( 1, s.length( ) – 1 ) );
  }
}
    What would be the output of this function on the following parameters?
    1. enigma( “Hi” )
    2. enigma( “Homer” )
    3. enigma( “Simpsons” )
  6. Consider the following recursive Java method: 
    public int mystery( int n ) {
  if ( n == 1 ) {
    return 2;
  }
  else {
    return n + mystery( n – 1 ) + mystery( n / 2 );
  }
}
    What does this method return for the following calls?
    1. mystery( 2 )
    2. mystery( 3 )
    3. mystery( 4 )
    4. mystery( 5 )
  7. Consider the following Java method: 
    public int riddle( int a, int b ) {
  if ( a == 0 )
    return 0;
  else if ( b == 0 )
    return a;
  else if ( b == 1 )
    return a * a;
  else
    return a * a * riddle(a, b–3);
}
    What would be the output of the following method calls? If it never finishes, write “never finishes”.
    1. riddle(4, 1)
    2. riddle(3, 4)
    3. riddle(2, 6)
    4. riddle(0, 5)
    5. riddle(5, 5)
  8. Write a recursive Java method named sumUp that takes one integer parameter n. This function should return the sum of all the integers from 1 to n. You may assume that n is 1 or larger. The function should be written recursively.
  9. Write a recursive Java method named stars that takes one integer parameter n. The function should return a string consisting of n stars. You may assume that n is 0 or larger. The function should be written recursively.
  10. Write a recursive Java method named reverse that takes one String parameter s. The function should return a string that is the reverse of s. This function should be written recursively.
  11. Consider the following Java method: 
    public void puzzle( int n ) {
  if ( n == 1 )
    System.out.println( “Hello” );
  else {
    System.out.println( “Hello” );
    morePrinting( n – 1 );
    puzzle( n – 2 );
  }
}
    Assuming that the important step in this algorithm is printing the word “Hello”, and assuming that calling morePrinting with a parameter of x results in printing 2x “Hello”s, write a recurrence equation (including the base case!) for this algorithm. You do not need to solve for T( n ) – just write the initial expression.
  12. Consider the following Java method: 
    public int unknown( int n ) {
  int x, y;
  if ( n == 1 )
    return 1;
  else {
    x = unknown( n / 3 );
    y = unknown( n / 2 );
    return x * y;
  }
}
    Assuming that the important step in this algorithm is a multiplication, write a recurrence equation (including the base case!) for this algorithm. You do not need to solve for T( n ) – just write the initial expression.
  13. Solve the following recurrence relation: 
    T( 1 ) = 2
T( n ) = T( n – 1 ) + 1
  14. Solve the following recurrence relation: 
    T( 1 ) = 1
T( n ) = 2T( n / 2 ) + n – 1
  15. Solve the following recurrence relation: 
    T( 2 ) = 1
T( n ) = 2T( n / 2 ) + n
